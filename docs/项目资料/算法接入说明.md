# 算法接入说明

本文说明用户算法包接入平台后的审核、运行和验收口径。

## 当前治理规则

- 用户在“算法接入”提交代码包。
- 管理员审核算法申请，可决定是否允许进入运行链路。
- 审核通过后，系统生成当前用户自己的 `UserPackage` 算法记录。
- 平台共享与本人可运行是两个动作：审核通过不等于自动全站开放。
- 管理员可将合适资源收录为平台算法，也可处理社区发布、举报和下架。

## 运行协议

算法包通过独立进程运行，后端不直接 `import` 用户代码。

推荐入口形式：

```bash
python infer.py --input <input_path> --output <output_path>
```

上传包支持：

- 单个 `.py` 文件：直接作为入口脚本执行。
- zip 包：优先寻找 `infer.py`；若只有一个 `.py` 文件，则使用该文件。

## 图像任务

已支持任务：

- `denoise`
- `deblur`
- `dehaze`
- `sr`
- `lowlight`

Worker 会准备输入图像和输出路径，算法输出结果图像后进入 PSNR、SSIM、NIQE 等指标计算链路。

## 视频任务

平台主线已支持：

- `video_denoise`
- `video_sr`

视频任务重点检查输出文件存在、可读，以及帧数/尺寸满足评测要求。视频算法接入应复用相同的审核、隔离执行、超时和失败回写机制。

## 失败路径

常见失败会写入 Run 详情和结构化错误信息：

- 算法包缺失
- 入口脚本缺失
- 脚本执行超时
- 脚本返回非零退出码
- 输出文件不存在或不可读
- 输出尺寸、帧数或任务类型不匹配

## 验收建议

1. 使用 `backend/tools/sample_algorithm_denoise.py` 或同协议脚本提交算法接入申请。
2. 管理员审核通过，并确认算法在用户算法列表中可见。
3. 在 `NewRun` 选择该算法与真实数据集创建任务。
4. 确认 Run 状态为 `done`，指标为真实计算值，详情中存在运行记录。
5. 删除申请或算法记录时，确认关联记录按当前业务规则同步清理。
