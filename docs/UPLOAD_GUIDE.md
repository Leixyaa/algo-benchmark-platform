# 资料如何提供给 AI（毕设开题/任务书/学校规范）

## 推荐方式（最省事、最稳）

### 方式 A：把文件放进仓库里（建议）
1. 在项目根目录创建 `docs/graduation/`（如果已经存在就直接用）。
2. 将资料文件放入该目录，例如：
   - `开题报告.md` / `开题报告.txt`
   - `任务书.md` / `任务书.txt`
   - `学校毕设要求.md` / `学校毕设要求.txt`
3. 在对话里告诉我文件路径（例如 `docs/graduation/开题报告.md`），我会直接读取并提取关键信息。

### 方式 B：直接复制粘贴正文（适合短文档）
把关键章节粘贴到对话里即可。建议按以下顺序粘贴：
1. 课题背景与意义
2. 研究目标/任务
3. 主要功能模块与范围边界
4. 技术路线/算法与指标
5. 进度安排与验收要求

### 方式 C：截图/拍照（适合 PDF/扫描件）
直接把页面截图发来即可。优先发：
- 任务目标、验收指标、功能要求、格式规范、查重/引用要求等页

## 不太推荐（但也能用）
- 仅上传 PDF/DOCX 而不提供可复制文本：我可能无法稳定提取内容，建议同时导出一份 `.md` 或 `.txt` 放到 `docs/graduation/`。

## 隐私与脱敏建议
- 可删除/打码：学号、身份证号、手机号、家庭住址、邮箱等个人信息。
- 指导老师姓名、学院/专业通常无需脱敏（按你学校要求）。

## 我收到资料后会做什么
- 把关键约束（功能范围、验收指标、格式规范）整理进 `AGENTS.md`
- 生成可执行的开发清单（按模块拆分）
- 在每次改完代码后同步更新进度到 `AGENTS.md`
