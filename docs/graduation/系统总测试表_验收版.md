﻿# 系统总测试表（验收版）

## 1. 使用说明
- 目标：让不了解项目的人也能按表完成全量测试，并形成可验收证据。
- 测试范围：前端页面、后端 API、异步执行链路、核心算法、导出能力、异常路径、真实性与稳定性。
- 执行方式：按“测试编号”从小到大执行；每条测试都要留证据（截图/导出文件/日志）。
- 结论规则：单条测试“通过/失败”；总验收按第 7 节判定。

## 4. 真实性与可用于真实场景的测试

| 编号 | 检查点 | 操作流程 | 预期结果 | 原理说明 | 通过标准 |
|---|---|---|---|---|---|
| R-01 | 数据真实性 | 使用真实采集数据集执行同类任务 3 次 | 指标存在波动但在合理区间 | 真实数据不可“完美稳定”，应有自然波动 | 曲线不异常、无固定伪值 |
| R-02 | 指标合理性 | 人工抽查输出图像质量与 PSNR/SSIM/NIQE | 视觉趋势与指标趋势一致 | 指标需与主观质量方向一致 | 至少 5 组样本一致 |
| R-03 | 重复性 | 固定同参数重复运行同任务 | 结果相近，误差在可接受范围 | 验证系统可复现 | 指标偏差可解释 |
| R-04 | 稳定性 | 连续批量创建 10+ Run | 系统无崩溃，无大面积 stuck | 真实场景下需支持持续任务 | 完成率高、错误可解释 |
| R-05 | 可追溯性 | 查看 run 详情与导出记录 | 参数、指标、耗时、错误可追溯 | 验收核心是“可审计” | 任一 Run 可完整复盘 |

## 5. 异常与边界测试（防出错）

| 编号 | 异常场景 | 操作流程 | 预期结果 | 通过标准 |
|---|---|---|---|---|
| E-01 | 数据集不存在 | 使用不存在 dataset_id 创建 Run | 返回失败与明确提示 | 错误消息可读、无 500 崩溃 |
| E-02 | 算法任务不匹配 | 去雾任务选择去噪算法 | 创建被拒绝 | 返回 task mismatch 类提示 |
| E-03 | 参数对象非法 | 通过预设/方案注入非法参数对象后创建 Run | 前后端均阻断并提示 | 不发送或不落地错误参数 |
| E-04 | 导出空结果 | 筛选到 0 条后导出 | 给出“无数据”提示 | 不生成空白错误文件 |
| E-05 | 取消重复点击 | 对同一 Run 多次取消 | 系统幂等处理 | 不出现异常状态抖动 |

## 6. 新手执行顺序（建议）
1. 执行 P-01 ~ P-04（环境与数据准备）  
2. 执行 F-01 ~ F-07（数据集与算法准备）  
3. 执行 F-08 ~ F-14（Run 主链路与导出）  
4. 执行 F-15 ~ F-21（Compare、推荐导出与批量基线）  
5. 执行 F-22 ~ F-25（覆盖与错误体系）  
6. 执行 F-26 ~ F-27（算法真实接入）  
7. 执行 R-01 ~ R-05 + E-01 ~ E-05（真实性与稳定性）
