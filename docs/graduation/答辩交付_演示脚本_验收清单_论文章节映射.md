# 答辩交付包：演示脚本、验收清单、论文章节映射

## 1. 演示脚本（10~12分钟）

### 1.1 开场（第 0~1 分钟）

- 课题目标：解决算法评测口径不统一、复现成本高、选型效率低。
- 系统主线：数据集管理 → Run 异步执行 → 指标评估 → 对比推荐 → 导出归档。

### 1.2 系统结构讲解（第 1~3 分钟）

- 展示架构图：Web / FastAPI / Redis / Celery Worker。
- 说明计算环境：系统同时具备 Web 端与桌面端两种运行形态；桌面端由本地静态前端、独立启动脚本与本地 SQLite 数据文件组成。
- 强调工程亮点：异步任务、结构化错误码、取消一致性、可复现 record。
- 对照文档：`docs/graduation/系统架构_流程_ER_实现说明.md`。

### 1.3 功能演示（第 3~9 分钟）

- **步骤A 数据集**
  - 打开 Datasets 页面，演示 scan 后 `task_types` / `supported_task_types` / `pairs_by_task`（与后端同步后的 meta）。
  - 说明新建数据集可不强制手填任务类型；发布社区前「已识别任务」校验。
  - 说明同名配对规则与 `strict_validate` 的作用。
  - 社区列表：`scope=community` 排除 `owner_id=system`，并对候选集再校验公开可用。
- **步骤B 算法接入**
  - 打开“算法接入”，展示固定协议：`python infer.py --input <input_image> --output <output_image>`。
  - 说明“审核通过仅生成本人用户算法，管理员另行收录才会上平台算法”的治理口径。
- **步骤C 发起 Run**
  - 在 NewRun **两步**（基础配置 → 算法选型）选择任务、数据集、多选算法与指标；无「参数方案」下拉，统一默认参数；可选多算法保存预设。
  - 数据集下拉按**当前任务类型**过滤（与 `datasetTask.js` 一致）。
  - 展示内置算法与本人已接入 `UserPackage` 算法都可被选择；新 Run 经历 queued/running/done。
- **步骤D 结果与对比**
  - 在 Runs 查看详情（metrics、record、error_code）。
  - 指出 `record.algorithm_runtime` 中的真实脚本执行留痕，证明跑的是上传代码而非模拟结果。
  - 在 Compare 选择同任务 run，对比排序与推荐原因。
- **步骤E 导出**
  - 导出 CSV/XLSX；导出 Compare 结论 Markdown。

### 1.4 主线收束（第 9~12 分钟）

- 覆盖任务：图像五类 + 视频两类。
- 覆盖能力：失败路径、重试、取消、导出一致性。
- 覆盖计算环境：浏览器 Web 端 + 本地桌面端。
- 结论：系统满足课程设计目标，具备继续扩展算法的工程基础。

## 3. 故障预案（答辩现场）

### 3.1 Worker 无响应

- 现象：Run 一直 queued。
- 快速处理：检查 Redis 连通；重启 Worker；使用 `celery inspect ping` 确认节点在线。

### 3.2 strict_validate 触发拒绝

- 现象：返回 `E_DATASET_NO_PAIR`。
- 快速处理：检查输入目录与 `gt/` 文件同名；重新 scan。

### 3.3 取消不收敛

- 现象：canceling 偶发被 done 覆盖。
- 说明点：系统已加 run 内标记 + `run_cancel:{run_id}` 双保险；现场可演示 queued 与 running 的 cancel。

### 3.4 导出异常

- 现象：导出文件为空或字段不全。
- 快速处理：先筛选 `status=done`，再导出；确认 runs 存在 metrics/record。

### 3.5 桌面端入口异常

- 现象：桌面窗口未打开或页面未同步到最新版本。
- 快速处理：重新执行 `scripts/manual_up_desktop.ps1`；必要时先执行 `npm run build:desktop` 或 `scripts/build_desktop_bundle.ps1` 更新桌面构建产物与交付包。

## 4. 论文章节映射（可直接落稿）

### 第1章 绪论

- 问题背景：评测口径不统一、复现成本高、选型效率低。
- 目标与贡献：统一评测流程 + 可观测异步执行 + 可导出对比推荐。

### 第2章 需求分析

- 功能需求：数据集管理、算法管理、Run、Compare、导出。
- 运行环境需求：同时支持 Web 端与桌面端两种计算环境。
- 非功能需求：可复现性、稳定性、可扩展性、可追踪性。

### 第3章 系统设计

- 直接引用：架构图、流程图、ER 图（见“系统架构_流程_ER_实现说明.md”）。

### 第4章 系统实现

- 后端：FastAPI + Celery + Redis；错误码体系；取消一致性机制。
- 前端：Vue3 + Pinia + Element Plus；Runs/Compare 交互。
- 算法执行：图像五类 + 视频两类 baseline，以及图像 `UserPackage` 的受控真实接入。

### 第5章 测试与结果分析

- 覆盖 F/E/D/G3 验收项。
- 展示典型 run 指标、Compare 排序、导出样例。

### 第6章 总结与展望

- 已完成：主线闭环与工程稳定性增强。
- 展望：补齐视频算法接入、扩展更多指标、支持批量调参与自动报告模板化。
