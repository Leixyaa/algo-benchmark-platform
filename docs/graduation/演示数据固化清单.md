# 演示数据固化清单（防翻车）

> 目标：把“图像 + 视频”各 1 套演示数据固定下来，保证每次验收点击路径一致。

## 1. 固化规则

- 图像演示集：必须满足目标任务下 `可用配对数 > 0`（以扫描后的 `pairs_by_task` / 元数据为准）
- 视频演示集：必须满足 `video_denoise` 或 `video_sr` 下 `可用配对数 > 0`
- **任务类型与数据集匹配**：`发起评测`、`结果对比` 会按**当前所选任务类型**过滤数据集；请保证固化数据集在该任务下已被识别且配对>0（参见 `web/src/utils/datasetTask.js` 与数据集 `meta`）
- 每套数据至少提前跑通 1 次完整链路（发起评测 -> 任务中心 -> 结果对比）
- 固化后不再临时替换数据集，避免现场重扫或路径异常
- **社区数据**：公开社区列表已排除系统账号数据集；若演示社区上传，发布前需满足「已识别任务」等校验

## 3. 验收当天固定点击路径

1. `发起评测`：先走图像任务（固定任务 + 固定数据集 + 固定算法）
2. `任务中心`：展示状态与详情（创建/开始/完成时间）
3. `结果对比`：同任务同数据集筛选，展示推荐与导出
4. 再走视频任务同样流程（可简化为 1 次创建 + 1 次详情）

## 4. 现场兜底策略（按优先级）

- 兜底 A：如果视频数据集异常，先完整演示图像主链路，再说明视频已在预演通过
- 兜底 B：若某算法临时不可用，改用固定对比算法（提前在清单填好）
- 兜底 C：若 AI 模型未连通，演示本地 FAQ 与离线兜底回复
