# 计算环境条件一满足说明

## 结论

当前项目已具备两种可独立说明的计算环境：

1. Web 端
2. 桌面端

因此可按“至少包含两种计算环境”口径说明满足条件一。

## 计算环境一：Web 端

Web 端基于 Vue3 + Vite 实现，浏览器访问即可使用，核心入口包括：

- 数据集管理
- 算法管理
- 运行创建与状态跟踪
- 对比推荐与导出

对应代码位置：

- `web/src/views/`
- `web/src/router/index.js`
- `web/src/api/`

Web 端运行方式：

- 浏览器访问 `http://localhost:5173/`
- 调用后端 API `http://127.0.0.1:8001/`

## 计算环境二：桌面端

桌面端不是简单的浏览器书签，而是具备独立启动链路的本地桌面运行形态，具体体现为：

- 有专门桌面入口脚本 `scripts/manual_up_desktop.ps1`
- 前端使用桌面专用构建产物 `web/dist-desktop/`
- 桌面模式前端通过本地文件入口运行，而不是依赖 `vite dev`
- 桌面端会启动本地后端 API 与 Celery Worker
- 桌面端默认使用本地 SQLite 数据库文件 `backend/data/desktop/abp-desktop.db`
- 有 Electron 桌面壳代码 `desktop/main.js`
- 在 Electron 不可用时，仍可通过 Edge app mode 以桌面窗口形态运行本地构建产物

对应代码位置：

- `desktop/main.js`
- `desktop/package.json`
- `scripts/manual_up_desktop.ps1`
- `web/.env.desktop`
- `web/vite.config.js`

桌面端运行方式：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\manual_up_desktop.ps1
```

## 为什么桌面端可作为独立计算环境

本项目桌面端满足以下特征：

- 具备独立启动入口
- 具备本地桌面窗口形态
- 具备本地构建前端资源
- 具备独立桌面运行配置
- 具备本地数据库落盘能力
- 不依赖开发态 `npm run dev` 页面作为正式入口

因此该桌面端可以区别于普通浏览器访问场景，与 Web 端形成两种不同的计算环境。

## 答辩建议表述

建议在答辩中这样表述：

“本系统同时支持 Web 端与桌面端两种计算环境。Web 端面向浏览器访问；桌面端基于独立桌面启动脚本和本地构建产物运行，默认使用本地 SQLite 数据库存储，能够以桌面应用方式完成数据集管理、算法评测、结果对比与导出等核心流程。”
