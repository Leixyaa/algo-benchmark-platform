# 资料关键信息摘录（已同步）

> 用途：把开题报告/任务书/学校规范中的硬性约束提取成“可落地的开发与验收清单”。  
> 资料来源：`docs/graduation/` 目录下的开题报告/任务书/学院要求文件 + 你在对话中给出的总结。

## 1. 课题信息
- 项目名称（对外）：算法性能测试与快速选型平台（本科毕设，系统开发类）
- 毕设题目（开题/任务书）：面向图像视频增强复原任务的开放式算法性能测试平台设计与开发
- 专业/方向：计算机科学与技术（系统开发类）

## 2. 系统目标与范围
- 系统要解决的问题：评测条件不统一、复现门槛高、结果对比不直观，导致算法选型靠经验/试错
- 核心目标：
  - 一站式评测：数据管理 → 发起评测 → 异步执行 → 指标计算 → 结果对比 → 报告导出
  - 统一口径：统一输入输出、预处理、指标与对比维度，保证可比性与可复现性
  - 快速选型：提供多指标的推荐/排序（加权或规则；可扩展为 Pareto 前沿等）
- 明确不做的内容（范围边界）：
  - 不做算法创新（核心是系统功能开发、工程化与完善）

## 3. 任务范围（覆盖的任务类型）
- 图像相关：
  - 去噪、去模糊、去雾、超分、低照度增强
- 视频相关：
  - 视频去噪、视频超分

## 4. 功能需求（按模块）
- 数据集管理：
  - 支持多数据集条目管理；建议支持图像/视频分类与任务类型标注
  - 支持数据上传/导入、预览、GT/输入对齐（全参考指标需要 GT）
- 算法库管理：
  - 算法条目增删改查；记录算法名称、任务类型、参数、版本/实现来源
  - 支持扩展接入新任务类别或新算法
- 评测任务（Run）：
  - 创建评测任务：选择任务类型 + 数据集 + 算法 + 参数/权重
  - 异步执行：队列调度；状态至少包含 queued → running → done（可扩展 failed/canceled）
  - 指标计算：PSNR、SSIM（全参考）+ NIQE（无参考）；建议记录耗时
- 结果展示与对比：
  - 按任务类型、数据集、算法筛选展示
  - 支持对比图表（表格 + 可视化），并输出推荐结果与理由
- 日志/导出/报告：
  - 导出对比结果与推荐原因（CSV/Excel）
  - 建议补充：任务运行耗时、错误信息、关键配置快照

## 5. 非功能需求与验收指标
- 可复现性：
  - 同一数据、同一算法、同一参数应得到可复现的指标结果（对随机过程需固定随机种子）
  - 评测记录需结构化保存：数据集信息、算法版本、参数、运行耗时、指标结果
- 可比性：
  - 算法对比必须在“同一评测协议”下进行（数据、预处理、指标计算一致）
- 稳定性：
  - 异步任务不阻塞 API；多任务情况下保持响应稳定
  - 失败任务可定位原因（日志/错误信息）
- 评估指标：
  - PSNR、SSIM、NIQE（现阶段核心指标集合）；后续可扩展更多指标
- 验收方式（毕设交付侧重）：
  - 系统可运行、流程清晰、演示顺利；功能模块完整；论文能准确反映设计与测试评估

## 6. 技术栈与环境（约束）
- 前端：Vue3 + Vite；Pinia；Element Plus；ECharts；Axios；LocalStorage 持久化
- 后端：FastAPI；Celery + Redis；Python 3.x；OpenCV / scikit-image / scikit-video
- 环境：Windows 开发；Redis 使用 Docker 容器

- 需在系统内最终上传材料：《任务书》《开题报告》《论文正文》《AI工具使用责任书》
- 开题报告字数：不少于 3000 字（并按学院模板）
- 过程管理记录本：至少填写 6 次《毕业设计指导记录表》，并需手签/电子签；学生全程携带，最终上交留档
- 参考文献数量要求（开题报告给出）：
  - 学术算法类：≥20 篇；外文 ≥8 篇；近三年 ≥5 篇
  - 纯项目开发类：≥15 篇；外文 ≥5 篇；近三年 ≥3 篇
- AI 工具使用（学校指导意见“试行”）：
  - 允许：文献检索与整理、图表类型推荐与辅助制图、非创新性方法的代码辅助（需审核与测试）
  - 禁止：直接生成论文正文/开题报告等材料；虚构或篡改数据/图表；生成研究设计/创新性方案等
  - 责任：需征得指导教师同意；内容需核查；按要求提交披露/责任书；涉密内容禁止上传
